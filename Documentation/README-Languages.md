# README Languages

English is the default README surface for SwiftIntelligence.

This repository now ships `18` README language surfaces to reduce adoption friction for the largest and fastest-growing GitHub developer communities.

## Truthfulness Note

GitHub does not publish a direct official ranking of human languages used by all GitHub users.

The language set below is an inference based on official GitHub Octoverse country-community data:

- Octoverse 2025: `United States`, `India`, `China`, `Brazil`, `United Kingdom`, `Japan`, `Germany`, `Indonesia`, `Russia`, `Canada`, with strong momentum in `Egypt`, `Nigeria`, and `Morocco`
- Octoverse 2024: strong growth also highlighted across `Mexico`, `Colombia`, `France`, `Spain`, `Turkey`, and `Netherlands`

Sources:

- https://github.blog/news-insights/octoverse/octoverse-a-new-developer-joins-github-every-second-as-ai-leads-typescript-to-1/
- https://github.blog/news-insights/octoverse/octoverse-2024/

## Selection Logic

- `English` stays default because it is still the broadest cross-border developer language on GitHub.
- `Spanish`, `Portuguese`, `Arabic`, and `Chinese` cover multiple large GitHub markets, not only one country each.
- The remaining languages cover large top-10 GitHub countries or strong-growth developer regions highlighted by GitHub.
