# Getting Started with SwiftIntelligence

SwiftIntelligence currently ships as a modular Swift package for Apple platforms. Start by choosing only the products you need.

## Five-Minute Success Path

Use this path if you want the fastest real proof of value.

Target flow:

`Vision -> NLP -> Privacy`

Target demo:

- [IntelligentCameraApp.swift](../Examples/DemoApps/IntelligentCamera/IntelligentCameraApp.swift)

Why this path:

- it is the strongest maintained flagship flow in the repo
- it proves multi-module composition rather than one isolated wrapper
- it aligns with the current showcase and release-proof story

Step 1. Add the smallest flagship dependency set:

```swift
dependencies: [
    .package(url: "https://github.com/muhittincamdali/SwiftIntelligence.git", from: "1.0.0")
],
targets: [
    .target(
        name: "MyApp",
        dependencies: [
            .product(name: "SwiftIntelligenceCore", package: "SwiftIntelligence"),
            .product(name: "SwiftIntelligenceVision", package: "SwiftIntelligence"),
            .product(name: "SwiftIntelligenceNLP", package: "SwiftIntelligence"),
            .product(name: "SwiftIntelligencePrivacy", package: "SwiftIntelligence")
        ]
    )
]
```

Step 2. Configure the core once:

```swift
import SwiftIntelligenceCore

SwiftIntelligenceCore.shared.configure(with: .production)
```

Step 3. Walk the flagship proof surface:

- [Showcase](Showcase.md)
- [Examples Status](../Examples/README.md)
- [Intelligent Camera demo guide](../Examples/DemoApps/IntelligentCamera/README.md)
- [Intelligent Camera demo source](../Examples/DemoApps/IntelligentCamera/IntelligentCameraApp.swift)

Step 4. Verify locally:

```bash
swift build
bash Scripts/validate-flagship-demo.sh
bash Scripts/validate-examples.sh
swift test
```

What success looks like:

- top labels are populated from `Vision`
- OCR text appears from `analyzeDocument`
- summary is generated by `NLP`
- privacy preview shows tokenized output instead of raw extracted text

## Best First Paths

Use the smallest path that matches your first success:

- NLP-first app:
  `SwiftIntelligenceCore` + `SwiftIntelligenceNLP`
- Vision-first app:
  `SwiftIntelligenceCore` + `SwiftIntelligenceVision`
- Speech-first app:
  `SwiftIntelligenceSpeech`
- Privacy-aware AI flow:
  `SwiftIntelligencePrivacy` plus the feature module it protects

## Choose This Package When

- you want one maintained Swift package path across multiple Apple AI frameworks
- you expect to chain modules like `Vision -> NLP -> Privacy`
- you want examples, proof surfaces, and release validation in the same repo

## Stay On Raw Apple APIs When

- you only need one framework and already know its direct API surface well
- you need the thinnest possible wrapper layer for a single feature
- you are benchmarking Apple primitives directly rather than evaluating a higher-level toolkit workflow

If you are deciding between this package and raw Apple APIs, start here:

- [NLP vs NaturalLanguage](Comparisons/NLP-vs-NaturalLanguage.md)
- [Vision vs Apple Vision](Comparisons/Vision-vs-AppleVision.md)
- [Speech vs Apple Speech](Comparisons/Speech-vs-AppleSpeech.md)
- [Privacy vs CryptoKit + Security](Comparisons/Privacy-vs-CryptoKit-Security.md)
